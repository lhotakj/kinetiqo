Welcome to Kinetiqo

A powerful, containerized tool for synchronizing your Strava activities to any database. Built for developers who need reliable, scalable activity data management.

Powered by Strava API

Overview

Kinetiqo is a containerized application designed to seamlessly synchronize your Strava activities with any database backend. Whether you're building analytics dashboards, training logs, or custom fitness applications, Kinetiqo provides a reliable bridge between Strava and your data infrastructure.

Installation

Get started with Kinetiqo in minutes using Docker:

# Pull the latest image
docker pull kinetiqo/kinetiqo:latest

# Or build from source
git clone https://github.com/lhotakj/kinetiqo.git
cd kinetiqo
docker build -t kinetiqo .

Quick Start

Set up your environment and start syncing:

# Create a .env file
cat > .env << EOF
STRAVA_CLIENT_ID=your_client_id
STRAVA_CLIENT_SECRET=your_client_secret
DATABASE_URL=postgresql://user:pass@host:5432/db
EOF

# Run the container
docker run --env-file .env kinetiqo/kinetiqo:latest

Configuration

Kinetiqo supports various configuration options through environment variables:

  • STRAVA_CLIENT_ID - Your Strava application client ID
  • STRAVA_CLIENT_SECRET - Your Strava application client secret
  • DATABASE_URL - Connection string for your database
  • SYNC_INTERVAL - How often to sync (default: 3600 seconds)

Sync Activities

Learn how to configure and run activity synchronization:

  1. Authenticate with Strava using OAuth 2.0
  2. Configure your database connection
  3. Run the sync command or schedule automatic syncs
  4. Monitor sync status and logs

Database Setup

Supported databases:

  • PostgreSQL
  • MySQL
  • SQLite
  • MongoDB

Docker Deployment

Deploy using Docker Compose for production environments:

version: '3.8'
services:
  kinetiqo:
    image: kinetiqo/kinetiqo:latest
    env_file: .env
    restart: unless-stopped
    volumes:
      - ./data:/app/data
